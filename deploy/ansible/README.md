# Deploy (appmire-hetz1)

Deploys the pluvio backend (api + worker) to the shared **appmire-hetz1** box,
which is already bootstrapped (docker, ufw, nginx, certbot) by the charging
deploy. So this playbook only ships the app and adds an nginx vhost — no Traefik
(the compose `prod` profile is left unused; the host nginx is the public entry).

```
cd deploy/ansible
ansible-playbook site.yml                 # app (build+run) + vhost + TLS
ansible-playbook site.yml --tags app      # rebuild/redeploy containers only
ansible-playbook site.yml --tags nginx    # vhost / TLS only
```

- Connects as the `ansible` user (key `~/.ssh/id_ed25519`).
- `--tags app` rsyncs `backend/`, renders `.env`, and `docker compose up -d
  --build` (api + worker; api on `127.0.0.1:8000`).
- No secrets/DB: `PLUVIO_*` config is public; the forecast `cache` volume is
  regenerated by the worker, so nothing to migrate.

## Cutover

Validate first under `pluvio-hetz1.appmire.be` (set `pluvio_domain` in
`group_vars/all.yml`), then point `pluvio.appmire.be` DNS at the box, flip
`pluvio_domain` back, and run `site.yml --tags nginx` to issue the prod cert.
